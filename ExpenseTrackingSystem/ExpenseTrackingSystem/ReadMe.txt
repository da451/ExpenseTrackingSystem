1) Для запуска приложения необходимо в ExpenseTrackingSystem.exe.config прописать строку подключения к БД. в формате
"Data Source = {путь к MyDB.sdf}"

В качестве базы данных использовалась MS SQL CE 4.0.

так же возможно прийдется установить триальную версию DevExpress 13.2

2) в приложении использовались следующие технологии:
 - WPF с реализацией паттерна MVVM (MVVMLight toolkit)
 - FluentNHibernate для доступа к БД

 3) Так же были использованы контролы DevExpress. Для приложения использовал триальную версию DevExpress 13.2

 4) Краткое описание проекта  DAL. 
 
 DAL используется как прослойка между базой данных и приложением.

 Реализована с помощью технологии Hibernate.

 Созданы следующие сущности:
	Expense - Затрата
	Tag - тип затраты
	User - пользователь
 Каждая Затрата и Тип привязаны к Пользователю.

 Так же в проекте реализованы паттерны  Repository и UnitOfWork

 5) Краткое описание проекта  ExpenseTrackingSystem. 

 Для создания UI использовалась технология WPF, а так же контролы DevExpress.

 Само приложение использует паттерн MVVM, который был реализован с помощью библиотек MVVMLight toolkit.

                                    Описание папок в проекте

 Converter - содержит 2 IValueConverter: 
	TextToTagConverter - преобразует текст в тип затраты;
	VisibilityConverter - преобразует значение true/false   Visibility.Visible/Visibility.Hidden;

 Extensions - содержит расширения классов Сущностей(nhibernate) и Моделей(model MVVM), для преобразования Сущностей в модели и наоборот

 Model - Модели для MVVM:
	 ExpenseModel -  модель для затрат;
	 ExpenseCollection - коллекция затрат, наследуемая от ObservableCollection;
	 TagModel - модель для типа затрат;
	 TagCollection - коллекция типов затрат, наследуемая от ObservableCollection;
	 UserModel - модель для пользователей;

Notifications.MessengerMessage - содержит коды для взаимодействия View и ViewModel

6) Краткое руководство по использованию:

При запуске приложения открывается форма для ввода логина и пароля. 
Если у Вы еще не зарегистрированы нажмите линк "Регистрация". 
Логин и пароль может быть любым, нельзя использовать только пробелы.
В можно воспользоваться уже  созданным пользователем
логин: 1
пароль:1


После авторизации, открывается главная форма затрат.
На ней представлен список затрат с указанием даты, типа затрат, комментария и затраченной суммы.
Если необходимо удалить(изменить) затрату:
     1) выбираем затрату в списке
	 2) нажимаем на кнопку удалить(изменить)
	 3) выполняем изменение(удаление) выбранной затраты

Для добавления новой затраты нажимаем кнопку "new expense".
Появиться форма, на которой можно выбрать описание затраты(на этой форма можно так же создать новый тип затрат),
сумму, дату, комментарий.



Для просмотра всех типов затрат нажимаем кнопку "edit tags".
откроеться форма со списком типов.
- для добавления нового типа затрат вводим текст типа и нажимаем кнопку "new tag"
- для изменения уже существующего типа, выбираем нужный тип, изменяем его и нажимаем кнопку "save"



Для просмотра статистики, нажимаем кнопку statistics. 
Откроется форма на которой необходимо ввести промежуток дат, за которые будем смотреть статистику, и нажать 
кнопку Load.

Будет сформирован список, с группировкой по типам затрат, а так же круговая диаграмма, 
с указанием процента каждого типа затрат от общей суммы за выбранный период
